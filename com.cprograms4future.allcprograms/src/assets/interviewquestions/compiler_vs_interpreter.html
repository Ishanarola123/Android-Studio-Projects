Question:

Difference between Compiler and Interpreter

Answer:
Compiler Interpreter
Compiler translates entire program to machine code at once Interpreter translates line by line
Compiler generates intermediate code, called the object code or machine code Interpreter does not generate object code or machine code
More difficult to debug compared to Interpreter as all errors are shown at a time Easier to debug since execution will stop on finding the error in line
It takes large amount of time to analyze the source code but the overall execution time is comparatively faster. It takes less amount of time to analyze the source code but the overall execution time is slower.
Takes more memory compared to Interpreter as Intermediate code is genearated Takes Less Memory(Memory Efficient) as Intermediate code is not generated
Ex: C,C++,COBOL, Ex: BASIC,Ruby,Python
